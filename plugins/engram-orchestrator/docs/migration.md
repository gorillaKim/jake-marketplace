# engram-orchestrator v0.6.0 마이그레이션 가이드

이 문서는 기존 `engram-orchestrator` v0.5.x 버전을 사용하던 사용자가 v0.6.0 버전으로 안전하게 전환할 수 있도록 마이그레이션 가이드를 제공합니다.

---

## 1. 미션 레이어 도입 배경 및 DB 마이그레이션

v0.6.0에서는 **ADR-0014**에 따라, 스프린트와 에픽/이슈의 관계 구조가 아래와 같이 변경되었습니다:
- **기존 구조**: Sprint → Epic → Issue → Task
- **신규 구조**: Sprint → Epic (Sprint SSOT) 및 Mission (sprint-agnostic) → Issue (Epic으로부터 상속)

이에 따라 데이터베이스 스키마가 업데이트되었으며, 기존 데이터에 대한 백필 및 컬럼 제거가 내부 마이그레이션(`0008` ~ `0010`)을 통해 앱 실행 시 자동으로 완료됩니다. 사용자는 별도의 DB 쿼리나 작업을 수동으로 수행할 필요가 없습니다.

---

## 2. API 및 파라미터 변경 사항 (주의)

### `issue_create`에서 `sprint_id` 및 `mission_id` 인자 제거
이슈 단독으로 스프린트를 직접 지정하던 방식에서 탈피하여 에픽의 스프린트를 그대로 따르도록 규정이 강화되었습니다.
- **AS-IS (v0.5.x)**: 
  이슈 생성 시 `issue_create(epic_id, sprint_id, title, ...)`와 같이 스프린트를 직접 지정.
- **TO-BE (v0.6.0)**:
  이슈 생성 시 `issue_create(epic_id, title, ...)` 형식으로 스프린트와 미션 인자를 전달하지 않습니다. 이슈는 에픽으로부터 스프린트와 미션 소속을 **자동 상속**받습니다.
  
따라서 에이전트(예: `engram-analyzer`)가 이슈를 생성할 때는 에픽을 먼저 생성하고 그 에픽에 스프린트 및 미션을 연결해야 합니다.

---

## 3. 신규 스킬 `mission-plan` 활용

v0.6.0에는 대형 피처나 로드맵을 일괄적으로 계획 및 수립할 수 있는 `mission-plan` 스킬이 도입되었습니다.

- **트리거 키워드**:
  - `"미션 계획"`, `"로드맵"`, `"분기 목표 설정"`, `"마일스톤 계획"`
- **동작 방식**:
  1. 현재 활성 스프린트 및 미션 현황 파악.
  2. `mission_create`를 호출하여 최상위 미션을 신규 수립.
  3. `engram-analyzer`를 호출해 해당 미션 산하에 여러 에픽을 수립하고, 하위 이슈와 태스크들을 자동으로 계획하여 계층 구조로 연동합니다.

---

## 4. 스킬 및 하네스 피드백(EVALUATION) 및 대화형 회고 시스템 도입

v0.6.0에서는 에이전트가 engram을 더 영리하게 활용하고 개발 프로세스를 자가 개선할 수 있도록 피드백/회고 시스템이 강화되었습니다.

### 에이전트 작업 시 EVALUATION 피드백 작성 의무화
- **작업 수행자 (`engram-worker`, `solo-track`)**:
  - 작업을 완수하고 데모 상태로 전이하기 전에, 현재 실행 중인 하네스(Harness) 속도/안정성, 작업에 사용한 스킬(Skill)의 적절성, 그리고 Engram 사용성(토큰 소모량 등)을 평가하는 피드백 노트를 `note_type="reference"` (접두사 `[EVALUATION]`)로 작성해야 합니다.
  - `WORKER_RESULT` YAML의 `evidence` 내에 `evaluation_note_added: true` 필드가 필수로 보고되며, leader 에이전트가 실제 작성을 검증합니다.

### 스프린트 회고 리포트 및 대화형 회고 연동
- **회고 생성기 (`engram-retro`, `sprint-retro`)**:
  - 회고 리포트 생성 시 스프린트 내 누적된 `[EVALUATION]` 피드백 노트를 요약하여 **"10. 스킬 및 하네스 피드백"** 섹션을 자동으로 그립니다.
  - 회고 리포트 작성을 마친 후, 에이전트가 피드백을 기반으로 사용자와 상호작용 회고 대화(Retrospective Discussion)를 리드하며 프로세스 개선 액션 아이템 수립을 유도합니다.

---

## 5. 업데이트 방법

플러그인을 업데이트하려면 Claude Code 세션에서 아래 명령을 실행하거나 CLI를 재설치합니다:

```bash
# 플러그인 업데이트 후 Claude Code 재시작
engram hook uninstall && engram hook install
```
또는 데스크톱 Tauri 앱을 최신 버전으로 실행하면 백엔드 도구들이 최신 스키마 사양에 자동으로 맞춰집니다.
