# GTM Sync — 이벤트 정의 → GTM 컨테이너 자동 동기화

이벤트 정의 파일(`*.events.ts`)의 `defineEvents()` 구조를 파싱하여 GTM 컨테이너에 Data Layer 변수, Custom Event 트리거, GA4 Event 태그를 자동 생성합니다.

## Prerequisites

- Stape GTM MCP 서버가 `.mcp.json`에 등록되어 있어야 함
- Google OAuth 인증 완료 (최초 실행 시 브라우저 인증)

## Input

- `$ARGUMENTS`: 대상 이벤트 파일 경로 (빈 값이면 `src/events/*.events.ts` 전체)

## Workflow

### Phase 0: 사전 확인

1. GTM MCP 서버 연결 확인 — `gtm_account` 도구로 `list` 호출
   - 실패 시 → 사용자에게 OAuth 인증 안내 (브라우저에서 Google 로그인 필요)
2. `src/events/*.events.ts` 파일 스캔 → 사용 가능한 이벤트 파일 목록 수집

### Phase 1: 사용자 설정 수집

AskUserQuestion으로 다음을 확인:

```yaml

      - label: '기존 컨테이너 선택'
```

GTM 계정이 여러 개면 추가 질문으로 계정 선택.

**기존 컨테이너 선택 시 추가 플로우:**

1. `gtm_container` → `list` (선택한 accountId)로 컨테이너 목록 조회
2. AskUserQuestion으로 컨테이너 목록을 동적 옵션으로 표시하여 선택받기
   - 각 옵션: `label: "{컨테이너이름} ({publicId})"`, `description: "ID: {containerId}"`

### Phase 2: 이벤트 파일 파싱

1. 대상 `*.events.ts` 파일을 Read로 읽기
2. `defineEvents(prefix, schema)` 구조에서 추출:
   - **이벤트 목록**: `{prefix}_{domain}_{action}` 형식의 전체 이벤트명
   - **파라미터 목록**: 모든 고유 파라미터명 + 타입 (중복 제거)
   - **파일별 파라미터 매핑**: 어떤 파라미터가 어떤 파일에서 사용되는지 추적
3. 파싱 결과 요약을 사용자에게 제시:
   - 총 이벤트 수, 총 고유 파라미터 수
   - 센터별 이벤트 수
   - 공유 파라미터 수 (여러 파일에서 사용되는 파라미터)
4. 사용자 승인 요청

```yaml
questions:
  - question: '위 범위로 GTM 리소스를 생성할까요?'
    header: '작업 승인'
    options:
      - label: '승인, 진행'
        description: '위 범위대로 GTM 리소스 생성 시작'
      - label: '취소'
        description: '작업 취소'
    multiSelect: false
```

### Phase 3: GTM 리소스 생성

**반드시 순차 실행** (각 단계의 ID가 다음 단계에 필요):

**중복 방지**: 기존 컨테이너 모드에서는 새 워크스페이스를 생성하므로 기존 리소스와 충돌하지 않는다. 새 컨테이너 모드에서는 빈 컨테이너이므로 중복 없음. 동일 워크스페이스에서 2회 실행 시 중복 생성될 수 있으므로, 실행 전 기존 변수/트리거/태그를 `list`로 조회하여 이름 기준으로 이미 존재하면 스킵한다.

#### Step 3-1: 계정 확인

- `gtm_account` → `list` 호출
- 사용자가 선택한 accountId 확인

#### Step 3-2: 컨테이너 생성 또는 선택

- **새 컨테이너 모드**: `gtm_container` → `create`
  - `accountId`: Step 3-1에서 확인
  - `createOrUpdateConfig`: `{ "name": "xpert-da-web-YYYYMMDD", "usageContext": ["web"] }`
  - 반환된 `containerId` 저장
- **기존 컨테이너 모드**: Phase 1에서 사용자가 선택한 `containerId` 사용

#### Step 3-3: 워크스페이스 설정

- **새 컨테이너 모드**: `gtm_workspace` → `list`로 Default Workspace의 `workspaceId` 저장
- **기존 컨테이너 모드**: `gtm_workspace` → `create`
  - `createOrUpdateConfig`: `{ "name": "sync-YYYYMMDD", "description": "Auto-generated by gtm-sync skill" }`
  - 반환된 `workspaceId` 저장

#### Step 3-4: 폴더 생성

이벤트 파일별 폴더 + 공유 리소스 폴더를 생성한다.

1. **`shared` 폴더**: 여러 파일에서 공통으로 사용되는 파라미터 변수 + GA4 Measurement ID 상수

   - `gtm_folder` → `create`: `{ "name": "shared" }`
   - ⚠️ GTM은 `_` 시작 폴더명을 거부하므로 반드시 `shared` 사용
   - `folderId` 저장

2. **이벤트 파일별 폴더**: 각 `*.events.ts` 파일마다 1개
   - `gtm_folder` → `create`: `{ "name": "{파일명}" }` (예: `bm-center`, `da-center`)
   - 각 `folderId` 저장

#### Step 3-5: Data Layer 변수 생성

파싱된 고유 파라미터별로 `gtm_variable` → `create`:

```json
{
  "accountId": "{accountId}",
  "containerId": "{containerId}",
  "workspaceId": "{workspaceId}",
  "createOrUpdateConfig": {
    "name": "dlv - {paramName}",
    "type": "v",
    "parameter": [
      { "type": "template", "key": "name", "value": "{paramName}" },
      { "type": "integer", "key": "dataLayerVersion", "value": "2" }
    ]
  }
}
```

GA4 태그 생성 옵션 선택 시 추가로 GA4 Measurement ID 상수 변수 생성:

```json
{
  "createOrUpdateConfig": {
    "name": "GA4 Measurement ID",
    "type": "c",
    "parameter": [
      { "type": "template", "key": "value", "value": "G-XXXXXXXXXX" }
    ]
  }
}
```

각 변수의 `variableId` 저장.

**변수 폴더 배정 (생성 후 이동):**
변수는 `parentFolderId`로 생성 시 폴더 배정이 되지 않으므로, 생성 완료 후 `gtm_folder` → `moveEntitiesToFolder`로 일괄 이동한다:

```json
{
  "action": "moveEntitiesToFolder",
  "accountId": "{accountId}",
  "containerId": "{containerId}",
  "workspaceId": "{workspaceId}",
  "folderId": "{folderId}",
  "variableId": ["{variableId1}", "{variableId2}", ...]
}
```

**폴더 배정 규칙:**

- 하나의 파일에서만 사용되는 파라미터 → 해당 파일 폴더
- 여러 파일에서 사용되는 파라미터 → `shared` 폴더
- GA4 Measurement ID → `shared` 폴더

#### Step 3-6: Custom Event 트리거 생성

각 이벤트별로 `gtm_trigger` → `create`:

```json
{
  "accountId": "{accountId}",
  "containerId": "{containerId}",
  "workspaceId": "{workspaceId}",
  "createOrUpdateConfig": {
    "name": "{eventName}",
    "type": "customEvent",
    "parentFolderId": "{folderId}",
    "customEventFilter": [
      {
        "type": "equals",
        "parameter": [
          { "type": "template", "key": "arg0", "value": "{{_event}}" },
          { "type": "template", "key": "arg1", "value": "{eventName}" }
        ]
      }
    ]
  }
}
```

**폴더 배정**: 해당 이벤트가 정의된 파일의 폴더 `folderId`

각 트리거의 `triggerId` 저장.

#### Step 3-7: GA4 Event 태그 생성 (선택)

GA4 태그 생성을 선택한 경우, 각 이벤트별로 `gtm_tag` → `create`:

```json
{
  "accountId": "{accountId}",
  "containerId": "{containerId}",
  "workspaceId": "{workspaceId}",
  "createOrUpdateConfig": {
    "name": "GA4 - {eventName}",
    "type": "gaawe",
    "parentFolderId": "{folderId}",
    "parameter": [
      {
        "type": "template",
        "key": "measurementIdOverride",
        "value": "{{GA4 Measurement ID}}"
      },
      { "type": "template", "key": "eventName", "value": "{eventName}" },
      {
        "type": "list",
        "key": "eventParameters",
        "list": [
          {
            "type": "map",
            "map": [
              { "type": "template", "key": "name", "value": "{paramName}" },
              {
                "type": "template",
                "key": "value",
                "value": "{{dlv - {paramName}}}"
              }
            ]
          }
        ]
      }
    ],
    "firingTriggerId": ["{triggerId}"]
  }
}
```

**폴더 배정**: 해당 이벤트가 정의된 파일의 폴더 `folderId`

### Phase 4: 검증 + 결과

1. `gtm_folder` → `list`로 생성된 폴더 확인
2. `gtm_variable` → `list`로 생성된 변수 확인
3. `gtm_trigger` → `list`로 생성된 트리거 확인
4. `gtm_tag` → `list`로 생성된 태그 확인 (GA4 태그 옵션 선택 시)
5. 결과 요약 출력:
   - 컨테이너 이름 / ID / Public ID
   - 워크스페이스 이름 / ID (기존 컨테이너 모드일 때 특히 중요)
   - 생성된 폴더 수 / 변수 수 / 트리거 수 / 태그 수
   - 폴더별 리소스 수 내역
   - 실패 항목 (있으면)
6. 실패 항목이 있으면 1회 재시도

**기존 컨테이너 모드 안내:**

> 새 워크스페이스 `sync-YYYYMMDD`에 리소스가 생성되었습니다.
> GTM UI에서 워크스페이스를 검토한 후 게시(Publish)하세요.
> URL: {tagManagerUrl}

## Rules

1. **MCP 도구 순차 호출**: accountId → containerId → workspaceId → folderId → variableId → triggerId 의존성 순서 준수
2. **Rate limit 대응**: 429 에러 시 5초 대기 후 재시도 (최대 3회)
3. **파라미터 중복 제거**: 여러 이벤트에서 같은 파라미터명이면 변수 1개만 생성
4. **이벤트명 포맷**: `{prefix}_{domain}_{action}` — defineEvents 구조에서 자동 생성
5. **GA4 Measurement ID**: GTM 상수 변수로 생성, 실제 값은 사용자가 나중에 GTM UI에서 수정
6. **폴더 배정**: 파일 전용 파라미터 → 파일 폴더, 공유 파라미터 → `shared` 폴더
7. **워크스페이스 분기**: 새 컨테이너 → Default, 기존 컨테이너 → 새 워크스페이스 생성
