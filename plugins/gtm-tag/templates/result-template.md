## 기획자 참고 사항

### dataLayer 이벤트명과 GA4 이벤트명 관계

이 프로젝트에서 dataLayer에 push되는 이벤트명에는 팀 prefix가 포함됩니다.
GTM 트리거에서 prefix를 제거하여 GA4 이벤트명으로 매핑해야 합니다.

(project-config.md의 그룹 매핑에서 prefix별 행 동적 생성)

| prefix | 센터 | GTM 트리거 매칭 패턴 |
|--------|------|---------------------|
| ... | ... | ... |

예시:
- 코드에서 push: `da_ads_campaign_manage_click`
- GTM 트리거: 이벤트명이 `da_` 로 시작하는 커스텀 이벤트
- GA4 이벤트: `ads_campaign_manage_click` (prefix 제거)

### Host App 공통 변수

다음 변수는 Host App이 페이지 진입 시 dataLayer에 자동 push합니다.
개별 이벤트 파라미터에는 포함되어 있지 않으며, GTM 컨테이너에서 데이터 영역 변수로 참조하면 됩니다.

## GTM 컨테이너 설정 가이드

### Step 1: 데이터 영역 변수 생성

각 이벤트 파라미터를 GTM에서 사용하려면 **데이터 영역 변수**를 생성해야 합니다.

1. GTM 컨테이너 → **변수** → **사용자 정의 변수** → **새로 만들기**
2. 변수 유형: **데이터 영역 변수**
3. 설정:

| 변수 이름 (GTM) | 데이터 영역 변수 이름 | 용도 |
|-----------------|---------------------|------|
| `dlv - tab_name` | `tab_name` | 현재 탭명 |
| `dlv - search_keyword` | `search_keyword` | 검색어 |
| (이벤트별 파라미터마다 1개씩 생성) | | |

> Host App 공통 변수(project-config.md 참조)도 동일한 방식으로 데이터 영역 변수를 생성하면 모든 태그에서 참조할 수 있습니다.

### Step 2: 트리거 생성

이벤트별로 또는 센터 단위로 트리거를 생성합니다.

**방법 A: 이벤트별 개별 트리거** (정밀 제어)

1. GTM → **트리거** → **새로 만들기**
2. 트리거 유형: **맞춤 이벤트**
3. 이벤트 이름: `da_ads_campaign_manage_click` (dataLayer 이벤트명 그대로)
4. 이 트리거는 **일부 맞춤 이벤트**에서 실행 → Event equals `da_ads_campaign_manage_click`

**방법 B: 센터 단위 트리거** (간편, GA4 이벤트 태그에서 이벤트명 동적 매핑)

1. 트리거 유형: **맞춤 이벤트**
2. 이벤트 이름: `da_.*` (정규식 사용 체크)
3. 이 트리거는 `da_` prefix로 시작하는 모든 이벤트에서 실행

### Step 3: GA4 이벤트 태그 생성

1. GTM → **태그** → **새로 만들기**
2. 태그 유형: **Google 애널리틱스: GA4 이벤트**
3. 측정 ID: `{GA4 측정 ID}`
4. 이벤트 이름 설정:

**개별 트리거 사용 시** (방법 A):
- 이벤트 이름: `ads_campaign_manage_click` (prefix 제거한 GA4 이벤트명 직접 입력)

**센터 단위 트리거 사용 시** (방법 B):
- 이벤트 이름에 **맞춤 자바스크립트 변수** 사용하여 prefix 자동 제거:

\`\`\`javascript
// GTM 변수 이름: "cjs - strip prefix"
// 변수 유형: 맞춤 자바스크립트
function() {
  var eventName = {{Event}};
  // prefix 제거 (사용 중인 prefix 목록에 맞게 수정)
  // project-config.md의 prefix 목록으로 정규식 동적 생성. 예: prefix가 da, bm이면 /^(da|bm)_/
  return eventName.replace(/^({prefix1}|{prefix2})_/, '');
}
\`\`\`

5. 이벤트 매개변수 추가:

| 매개변수 이름 | 값 |
|-------------|-----|
| `tab_name` | `{{dlv - tab_name}}` |
| `search_keyword` | `{{dlv - search_keyword}}` |
| (이벤트에서 사용하는 파라미터만 추가) | |

6. 트리거: Step 2에서 생성한 트리거 연결

### Step 4: 미리보기로 검증

1. GTM → **미리보기** 클릭
2. 대상 사이트 URL 입력 후 연결
3. Tag Assistant에서:
   - **Tags Fired** 목록에서 태그 실행 확인
   - **Data Layer** 탭에서 push된 이벤트와 파라미터 확인
   - **Variables** 탭에서 데이터 영역 변수 값 확인

## dataLayer 테스트 방법

브라우저 콘솔에서 다음으로 이벤트 발행을 확인할 수 있습니다:

\`\`\`javascript
// dataLayer 모니터링
window.dataLayer.filter(e => e.event && e.event.startsWith('{prefix}_'))

// 특정 이벤트 확인
window.dataLayer.filter(e => e.event === '{prefix}_{domain}_{action}')
\`\`\`

debug 모드(개발 환경)에서는 모든 이벤트가 console.log로도 출력됩니다.

## 작성 가이드

1. **기획자가 모르는 용어는 반드시 설명**: dataLayer, prefix, 트리거 등
2. **GTM 트리거 설정 가이드 포함**: prefix 기반 매칭 방법
3. **스킵 사유는 구체적으로**: 스킵된 그룹이 있으면 사유 명시
4. **dataLayer 테스트 코드 포함**: 기획자가 직접 확인할 수 있도록
5. **이벤트 순서는 CSV 원본과 동일하게**: 기획자가 대조하기 쉽도록
